Chapter 5 监督学习

参考: 1. https://rpubs.com/liam/mlr3Tutorial mlr3教程

有监督学习是机器学习的一个主要分支，它涉及从标记的数据中学习模型，以预测或分类新的数据点。以下是有监督学习的概述：

  1. 标记数据：有监督学习依赖于标记的训练数据。这些数据包括特征（输入）和相应的标签（输出）。特征是描述数据点的属性，标签是我们要预测或分类的目标。

  2. 目标：在有监督学习中，我们的目标是使用训练数据构建一个模型，以便在未知数据上进行准确的预测或分类。这个目标可以是回归（连续值预测）或分类（离散类别标签）。

  3. 模型构建：有监督学习中的模型构建是学习算法的过程，它通过分析训练数据来建立模型。常见的有监督学习算法包括线性回归、决策树、支持向量机、神经网络、朴素贝叶斯、随机森林等。

  4. 特征工程：在有监督学习中，特征工程是一个重要的环节，它包括选择合适的特征、进行特征预处理、处理缺失值和异常值等。良好的特征工程可以显著影响模型性能。

  5. 训练和测试：训练数据集用于训练模型，测试数据集用于评估模型性能。通常，训练数据集占总数据的大部分，而测试数据集用于模型性能的验证。

  6. 评估指标：有监督学习使用各种评估指标来衡量模型的性能，如均方误差（Mean Squared Error）和准确度（Accuracy）。这些指标帮助我们了解模型的预测能力和泛化性。

  7. 泛化：有监督学习的主要目标是构建能够泛化到未见过的数据的模型。泛化能力是模型的一个关键属性，它衡量了模型在新数据上的预测效果。

  8. 超参数调优：许多有监督学习算法具有超参数，它们需要根据交叉验证等技巧进行调优，以获得最佳性能。

  9. 应用领域：有监督学习在各种领域中都有广泛的应用，包括自然语言处理、图像识别、金融风险分析、医疗诊断等。

总的来说，有监督学习是一种强大的机器学习方法，适用于许多现实世界的问题。通过构建模型并使用标记的数据进行训练，我们可以实现精确的预测和分类。

5.1 决策树

决策树（Decision Tree）是一种常用的机器学习算法，用于分类和回归问题。在R语言中，你可以使用rpart包来构建决策树模型，以及caret包来进行交叉验证和网格搜索以优化模型的超参数。

library(rpart)
library(caret)
## Loading required package: ggplot2
## Loading required package: lattice

我们将使用鸢尾花数据集，目标是预测鸢尾花的种类。

# 加载鸢尾花数据集
data(iris)

# 创建训练数据和测试数据
set.seed(123)  # 设置随机种子以确保结果可复现
trainIndex <- createDataPartition(iris$Species, p = 0.7, list = FALSE)
trainData <- iris[trainIndex, ]
testData <- iris[-trainIndex, ]

# 构建决策树模型
tree_model <- rpart(Species ~ ., data = trainData, method = "class")

交叉验证

# 创建交叉验证控制
ctrl <- trainControl(method = "cv", number = 10)

# 进行交叉验证
cv_results <- train(Species ~ ., data = trainData, method = "rpart", trControl = ctrl)

5.2 随机森林

随机森林（Random Forest）是一种强大的集成机器学习算法，它基于决策树构建，并通过随机选择数据和特征进行集成，以提高模型的性能和泛化能力。

library(randomForest)
library(caret)
# 加载鸢尾花数据集
data(iris)

# 创建训练数据和测试数据
set.seed(123)  # 设置随机种子以确保结果可复现
trainIndex <- createDataPartition(iris$Species, p = 0.7, list = FALSE)
trainData <- iris[trainIndex, ]
testData <- iris[-trainIndex, ]

# 构建随机森林模型
rf_model <- randomForest(Species ~ ., data = trainData)

使用caret包进行交叉验证来评估模型性能：

# 创建交叉验证控制
ctrl <- trainControl(method = "cv", number = 10)

# 进行交叉验证
cv_results <- train(Species ~ ., data = trainData, method = "rf", trControl = ctrl)

# 查看交叉验证结果
print(cv_results)

5.4 特征选择

参考资料:

  1. https://rpubs.com/liam/FSinR
  2. https://rpubs.com/liam/vip 变量重要性

在R语言中，有许多用于特征选择的方法，每种方法都有其适用的情境。

  1. 过滤法（Filter Methods）：过滤法通过对特征进行预处理，然后选择与目标变量具有高相关性的特征。
library(caret)

# 生成示例数据集（这里使用iris数据集）
data(iris)

# 将数据集划分为特征（X）和目标变量（y）
X <- iris[, 1:4]
y <- iris$Species

接下来，你可以使用过滤法来选择最相关的特征。在这个示例中，我们将使用方差分析（ANOVA）来计算特征与目标变量之间的关系，并选择具有显著差异的特征。

# 进行ANOVA分析
anova_result <- sapply(X, function(x) {
   print(summary(aov(x ~ y))[[1]][[5]][1])
})
## [1] 1.669669e-31
## [1] 4.492017e-17
## [1] 2.856777e-91
## [1] 4.169446e-85
# 设置显著性水平
alpha <- 0.05

# 选择显著性水平低于alpha的特征
selected_features <- names(anova_result[anova_result < alpha])

# 输出所选特征
cat("选择的特征：", selected_features, "\n")
## 选择的特征： Sepal.Length Sepal.Width Petal.Length Petal.Width
  1. 包装法（Wrapper Methods）：包装法使用特征子集来训练模型，然后选择效果最好的子集。
library(caret)
data(iris)
ctrl <- rfeControl(functions = rfFuncs, method = "cv", number = 10)
result <- rfe(iris[, 1:4], iris$Species, sizes = 1:4, rfeControl = ctrl)
selected_features <- result$optVariables
selected_features
## [1] "Petal.Width"  "Petal.Length"
  1. 嵌入法（Embedded Methods）：嵌入法将特征选择作为模型训练的一部分，如正则化。
library(glmnet)
## Loading required package: Matrix
## Loaded glmnet 4.1-6
data(iris)
x <- as.matrix(iris[, 1:4])
y <- as.numeric(iris$Species) - 1
fit <- cv.glmnet(x, y, family = "multinomial")
selected_features <- predict(fit, s = fit$lambda.min, type = "nonzero")
  1. 特征重要性（Feature Importance）：某些模型（如随机森林、XGBoost）提供了特征重要性评估，可以根据这些评估选择重要的特征。
library(randomForest)
data(iris)
rf_model <- randomForest(iris[, 1:4], iris$Species)
importance <- importance(rf_model)
selected_features <- rownames(importance)[importance > 0.1]
selected_features
## [1] "Sepal.Length" "Sepal.Width"  "Petal.Length" "Petal.Width"
